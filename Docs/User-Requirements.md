# User Requirements

## Must Have
- Use a free weather API.
- Display current temperature for the chosen city.
- Display a 5-day forecast.
- Handle loading and error states.
- Allow manual refresh.
- UI choice is up to the implementer.
- Any reasonable architecture is acceptable.
- Project should pass basic lint rules.

## Nice to Have (Optional)
- City search.
- City details screen with extra info from the API.
- Tests.

## Delivery Notes
- Timeline: 2 days from assignment.
- Expect to explain implementation and architectural decisions.
- 1-hour technical discussion in English after submission.
